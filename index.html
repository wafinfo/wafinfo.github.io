内网无密码认证远程直连MSSQL

发表于 2020-07-24 分类于 内网横向渗透

这里就不多介绍,为什么这里可以无密码就可以远程连接内网另外一台主机mssql数据库,直接上图(这里借用下百度照片之前的图没有保存)

实验环境

一台2003(域成员)主机装有2008sql server ip: 192.168.151.150
一台2008(域控)主机装有2008sql server ip: 192.168.151.155
cobaltstrike4.0 mimikatz.exe SocksCap64.exe

开始实验

这里起cobaltstrike4,前提我们获取了管理员用户权限,用mimikatz读取Administrator的hash,由于之前知道就没有截图,然后利用cs开启一个sock4a代理
这里我多说两句如果要进行内网扫描和爆破等等多线程操作都不要使用cs和msf代理,那个只适合代理工具进行访问,不稳定,不精确.



这边看Scokscap也代理成功了

这里把SSMS客服端加入到Scokscap里面,使他走隧道


接着，指定目标内网Mssql的ip，选择使用Windows 身份认证进行连接


至此已经无需密码成功连接.

实战也可以不用这么麻烦,当我们控制的主机如果也安装mssql,在能够远程登录情况下,可以使用Administrator用户直接进行远程登录,启用SSMS选择使用Windows 身份认证进行登录就行,
就可以省略hash注入那部操作,方便许多.

PTH的限制条件

发表于 2020-07-24 分类于 内网横向渗透

哈希传递攻击适用情况：

在工作组环境中：

Windows Vista 之前的机器，可以使用本地管理员组内用户进行攻击。

Windows Vista 之后的机器，只能是administrator用户(也就是SID为500,administrator改名也是一样只要SID为500即可)的哈希值才能进行哈希传递攻击，其他用户(包括管理员用户但是非administrator(SID为500))也不能使用哈希传递攻击，会提示拒绝访问。

在域环境中：

只能是域管理员组内用户(可以是域管理员组内非administrator用户)的哈希值才能进行哈希传递攻击，攻击成功后，可以访问域内任何一台机器。

kb2871997补丁
微软在2014年5月13日发布了针对Pass The Hash的更新补丁kb2871997,禁止本地管理员账户用于远程连接，这样就无法以本地管理员用户的权限执行wmi、PSEXEC、schtasks、at和访问文件共享
但是SID为500的用户不受kb2871997补丁影响,打了补丁主机,攻击方式一样有效如果非SID为500的用户在打了这个补丁就会无效.
