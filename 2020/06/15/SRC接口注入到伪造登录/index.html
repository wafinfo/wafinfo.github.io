SRC接口注入到伪造登录

发表于 2020-06-15 分类于 漏洞挖掘 阅读次数： Valine：

开局登录框,漏洞全靠爆.
这里接口从js里面找到,其实我后面扫了下目录发现也可以找到.

由于这里漏洞被修复就没办法截图了,漏洞报告里面看不清楚,就只能描述下,这里毫无防护直接丢进sqlmap跑

由于我询问了下审核人员这里能否深入,他说到表就行了,只能放弃从注入登录后台
在这个测出注入前我发现这个站是spring boot框架,测试发现存在Spring Boot Actuator未授权访问
这里简单讲下Springboot的actuator配置

/autoconfig 提供了一份自动配置报告，记录哪些自动配置条件通过了，哪些没通过
/configprops 描述配置属性（包含默认值）如何注入 Bean
/beans 描述应用程序上下文里全部的 Bean，以及它们的关系
/dump 获取线程活动的快照
/env 获取全部环境属性
/env/{name} 根据名称获取特定的环境属性值
/health 报告应用程序的健康指标，这些值由 HealthIndicator 的实现类提供
/info 获取应用程序的定制信息，这些信息由 info 打头的属性提供
/mappings 描述全部的 URI 路径，以及它们和控制器（包含 Actuator 端点）的映射关系
/metrics 报告各种应用程序度量信息，比如内存用量和 HTTP 请求计数
/metrics/{name} 报告指定名称的应用程序度量值
post /shutdown 关闭应用程序，要求 endpoints.shutdown.enabled 设置为 true（默认为 false）
/trace 提供基本的 HTTP 请求跟踪信息（时间戳、HTTP 头等）
/auditevents 显示应用暴露的审计事件 (比如认证进入、订单失败)
/loggers 显示和修改配置的loggers
/flyway 显示数据库迁移路径的详细信息
/liquidbase 显示Liquibase 数据库迁移的纤细信息
/scheduledtasks 显示应用中的调度任务
/threaddump 执行一个线程dump
/heapdump 返回一个GZip压缩的JVM堆dump
/logfile 返回log file中的内容(如果logging.file或者logging.path被设置)

http://xxxx/esc-sso/actuator/trace 运气也好蹲着一个员工未失效的cookie直接替换登录
